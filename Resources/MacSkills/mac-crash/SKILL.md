# 应用崩溃与无响应

1. 记录应用名、版本、崩溃时间、操作步骤和能否稳定复现。
2. 区分崩溃、无响应、系统强制终止和用户退出。
3. 读取对应时间的崩溃报告，关注异常类型、终止原因和崩溃线程，不凭一行堆栈定责。
4. 检查插件、权限、损坏配置、资源不足和版本兼容性。
5. 清除配置或重装前先备份用户数据并说明会丢失的状态。
6. 用相同步骤复现验证，不能只以“能打开”宣布修复。
